# Архитектурная записка

## Tender Document Generator

### Общая концепция

Решение представляет собой локальную систему автоматического заполнения тендерных документов на основе:
- **Входных данных**: JSON-файлы (profile.json, tender.json, calc.json)
- **Шаблонов**: DOCX-файлы с плейсхолдерами
- **Выходных данных**: Заполненные DOCX-документы

### Компоненты системы

#### 1. Data Layer (Входные данные)

**profile.json** — реквизиты компании:
- company: full_name, short_name, inn, kpp, ogrn, legal_address_full
- bank: name, account, correspondent_account, bik
- contact: responsible_name_full, email, phone
- signatory: position, name_short

**tender.json** — данные закупки:
- purchase_number, subject, offer_validity_days
- customer: данные заказчика
- items: список позиций номенклатуры

**calc.json** — расчёты:
- vat_rate, items (цены, количества)
- subtotal_wo_vat, vat_amount, total_with_vat

#### 2. Core Layer (Генератор)

**TenderDocumentGenerator** — основной класс:
- `__init__()` — загрузка данных из JSON
- `generate_all()` — последовательная генерация 3 документов
- `_generate_anketa()` — заполнение Анкеты
- `_generate_zayavka()` — заполнение Заявки
- `_generate_predlozhenie()` — заполнение Предложения о цене

#### 3. Output Layer (Результаты)

- 01_Анкета_участника_заполненная.docx
- 02_Заявка_на_участие_в_закупке_заполненная.docx
- 03_Предложение_о_цене_договора_заполненное.docx

### Логика заполнения

#### Детерминированная подстановка:
- Прямая замена плейсхолдеров на значения из JSON
- Заполнение таблиц по индексам строк и столбцов
- Форматирование дат и чисел по правилам

#### Обработка плейсхолдеров:
1. **Простые поля**: `[ИНН]` → `7705123456`
2. **Комбинированные**: `[Краткое наименование] [ИНН]` → `ООО «НордТехРесурс» 7705123456`
3. **Системные**: `[Дата в формате...]` → `«26» марта 2026 года`
4. **Табличные данные**: Позиции номенклатуры, итоговые суммы

### Технические детали

#### Используемые библиотеки:
- **python-docx** — работа с DOCX-файлами
- **json** — парсинг входных данных
- **pathlib** — работа с путями файловой системы
- **customtkinter** — GUI (опционально)

#### Форматы данных:
- Вход: JSON (UTF-8)
- Шаблоны: DOCX (Microsoft Word)
- Выход: DOCX

### Безопасность и изоляция

- ✓ Локальная обработка (без интернета)
- ✓ Нет внешних API-вызовов
- ✓ Нет логирования данных
- ✓ Синтетические данные в примерах

### Масштабируемость

Для добавления нового документа:
1. Добавить метод `_generate_<name>()`
2. Добавить вызов в `generate_all()`
3. Создать шаблон DOCX с плейсхолдерами
